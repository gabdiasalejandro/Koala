1 Plataforma DSL de Diagramas
Define una base común para describir conocimiento técnico con texto estructurado y renderizarlo en múltiples formatos.

core:: organiza -> 1.1 Núcleo del lenguaje
Gestiona sintaxis, parsing, normalización y validaciones mínimas para producir un modelo consistente.

syntax:: define -> 1.1.1 Gramática base
Establece reglas de numeración jerárquica, relaciones y bloques de contenido por nodo.

hl:: extiende -> 1.1.2 Extensiones por dominio
Permite agregar prefijos semánticos para representar áreas especializadas como bases de datos o arquitectura.

layout:: coordina -> 1.2 Motor de layout
Calcula posiciones y conexiones a partir del árbol lógico sin depender del formato final de salida.

layout:: incluye -> 1.2.1 Árbol top-down
Distribuye nodos de arriba hacia abajo con lectura secuencial clara para estructuras jerárquicas.

layout:: incluye -> 1.2.2 Cuadro sinóptico
Organiza nodos por columnas de izquierda a derecha para resumir categorías y subcategorías.

layout:: incluye -> 1.2.3 Diagrama radial
Coloca nodos por anillos según profundidad para resaltar relaciones alrededor de un concepto central.

render:: entrega -> 1.3 Capa de render
Transforma la escena geométrica en archivos visuales manteniendo estilos, tipografía y colores por tipo.

ui:: produce -> 1.3.1 Salida SVG
Genera un resultado editable e integrable en web, documentación y pipelines automáticos.

theme:: configura -> 1.3.2 Themes y tipografías
Permite variar color, contraste y jerarquía visual sin cambiar el contenido ni la geometría.

hl:: especializa -> 1.4 Dominio de base de datos
Representa conceptos de modelado relacional para comunicar diseño lógico y dependencias entre tablas.

hl:: modela -> 1.4.1 Entidades
Describe tablas principales del sistema con sus responsabilidades funcionales.

hl:: conecta -> 1.4.2 Relaciones
Explica vínculos entre entidades, direccionalidad y dependencias de integridad.

hl:: define -> 1.4.2.1 Cardinalidad
Indica reglas uno a uno, uno a muchos y muchos a muchos para cada relación.

hl:: restringe -> 1.4.2.2 Claves foráneas
Declara referencias entre tablas para mantener consistencia y trazabilidad de datos.

qa:: asegura -> 1.5 Calidad y evolución
Permite escalar el DSL sin regresiones visuales ni pérdida de compatibilidad con sintaxis previa.

rule:: valida -> 1.5.1 Reglas sintácticas
Aplica advertencias para títulos largos, profundidad excesiva y errores de conectores.

qa:: prueba -> 1.5.2 Pruebas visuales
Compara salidas por layout para asegurar estabilidad de estilos, conexiones y legibilidad.
